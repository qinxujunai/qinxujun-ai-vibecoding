# 提示词资产库 (Prompts Library)

这份文档保留了驱动这块项目的核心提示文本，它们可以被直接粘贴到各类 AI 对话框或者 CLI 代理的执行域中。

## 1. 项目审计提示词
**使用场景**: 让 AI 在接手陌生项目时迅速建立心理模型，而不至于上来就破环代码。
```text
请你作为一个资深前端工程师兼工程架构师，审计当前项目。
不要立即写代码！请先执行以下步骤：
1. 阅读 package.json 的依赖和脚本。
2. 查阅整体目录结构。
3. 根据已有代码猜测项目的设计规范和意图。
4. 输出一份 200 字以内的项目分析报告，并告诉我如果要在当前项目加 [功能名称]，你会修改哪些文件。
```

## 2. UI 截图审查提示词
**使用场景**: 搭配大模型视觉能力，比对 UI 和原设计的距离。
```text
请阅读随附的首屏 UI 截图，并对比业内顶尖产品（如苹果、Linear）官网的排版素养。
找出目前 UI 中存在的 3 处体验缺陷，重点考察：
1. 留白是否呼吸均匀？
2. 字体层级是否鲜明清晰？
3. 颜色是否显得高级而克制？
请给出明确修改代码的建议。
```

## 3. 安全部署前提示词
**使用场景**: 防止在开发后期直接部署带来的崩溃灾难。
```text
我准备部署。在部署前，请先在我本地执行一次 build：
1. 运行 `npm run build`。
2. 如果存在 TypeScript 类型报错，请指出并修复。
3. 检查是否有未使用的冗余组件。
4. 一切正常后，生成适合 git 提交的内容。
```
